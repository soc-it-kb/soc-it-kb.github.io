IT-SRV-SOP-LNX-01: Server Security Baseline — Linux

Minimum recommended security baseline for all Linux servers — ensuring systems are deployed and operated in a secure, resilient, and auditable state.

Section 1
Purpose

This SOP defines the minimum recommended security baseline for all Linux servers. The objective is to ensure Linux systems are deployed and operated in a secure, resilient, and auditable state, aligned with institutional security requirements and capable of supporting effective detection, response, and recovery.

Section 2
Scope
  • All supported Linux distributions (RHEL, Rocky, Alma, Ubuntu LTS, Debian)
  • On-premises, private cloud, and hosted Linux servers — physical and virtual
  • Servers supporting business, infrastructure, and security services

Out of scope: Linux workstations, network or security appliances where the OS cannot be managed.

Section 3
Minimum Security Requirements

💻 OS & Patch Management

  • Supported Linux distribution and version only
  • Security updates enabled or centrally managed
  • Unused packages and services removed
  • Kernel and system hardening parameters applied

🔐 Identity & Access

  • SSH access restricted to approved users and groups
  • Password-based SSH disabled where feasible
  • Key-based authentication enforced
  • Privilege escalation via sudo only — no direct root login
  • Shared accounts prohibited

🛡️ Hardening & ASR

  • Unnecessary services disabled
  • Network services bound only to required interfaces
  • SELinux or AppArmor enabled and enforcing where supported
  • File permissions restricted (least privilege)

📊 Logging & Telemetry

  • Authentication and privilege escalation logs enabled
  • System and security logs retained per policy
  • Logs forwarded to central monitoring platforms
  • Command execution logging for privileged users

🌐 Network Security

  • Host-based firewall enabled (iptables / nftables / firewalld)
  • Inbound traffic restricted to required services only
  • Administrative access restricted to management networks

🔒 Data Protection

  • Disk or volume encryption enabled where supported
  • Sensitive data protected at rest
  • Backups encrypted and protected
