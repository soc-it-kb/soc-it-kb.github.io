EP-04: Suspicious Process / LOLBin Activity

Answer each question YES or NO and apply the decision logic. Escalate with evidence attached — do not reboot or alter the endpoint before escalating.

💻 Endpoint threats Tier 1 Published 28 April 2026
🔔 When to use this checklist
EDR alert or analyst observation of suspicious process execution, abnormal parent-child process relationships, or known Living-off-the-Land Binary (LOLBin) abuse on a Windows endpoint.
PowerShell with encoded command cmd.exe spawned from Office app certutil.exe downloading file mshta.exe executing script Suspicious regsvr32 / rundll32 usage wmic.exe remote execution

Step 2 — Apply decision logic
✅
Single LOLBin execution, no network activity, no persistence — Document and escalate to Tier-2 for review. Context matters — confirm legitimate use case before closing.
⚠️
YES to questions 1, 3, 4, or 5 — Escalate to Tier-2 with full process tree. Do not reboot. Consider isolation if network activity is present.
🔴
YES to question 2, 6, or 8 — Escalate to Tier-3 immediately. Invoke EP-PB-02. Preserve volatile evidence. Isolate via EP-WI-02.
Step 3 — Capture evidence before escalating
Full process tree — parent, child, and grandchild processes
Complete command-line arguments including any encoded content
Process execution timestamps
Network connections made by the suspicious process
File paths — particularly any files written to temp or user directories
💡
Analyst Tip
Context is critical with LOLBins — PowerShell runs legitimately hundreds of times a day. The signals that matter are: unexpected parent process (Word spawning PowerShell), execution from temp directories, encoded commands, and network connections from processes that should never connect externally. Always capture the full command line before escalating.
