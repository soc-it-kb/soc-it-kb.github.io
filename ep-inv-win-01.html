EP-INV-Win-01: Windows Endpoint Investigation

Structured investigation guide for Windows workstations — covering initial triage through persistence analysis, network correlation, identity review, and final verdict. Includes a Tier-1 quick checklist followed by a full Tier-2 deep-dive investigation framework.

💻 Endpoint threats Windows workstations Tier 1 & 2 Tool-agnostic Published 27 April 2026

💡
Analyst Tip
Never reboot a suspected compromised endpoint before investigation — rebooting destroys volatile memory evidence (running processes, network connections, encryption keys) that cannot be recovered. Isolate first via EP-WI-02, then investigate.
🔍 Tier-2 Deep-Dive Investigation

📌 The following investigation framework applies once the endpoint is isolated or under observation. Work through each phase in order — do not skip phases. All findings must be documented in the incident record as you go.

1
Initial Triage & Context
Establish baseline
1
Confirm endpoint identity — hostname, asset tag, assigned user, department, and last seen time in EDR/asset inventory
2
Confirm isolation status — is the endpoint already isolated? If not, isolate now via EP-WI-02 before proceeding
3
Record the investigation start time, incident reference, and analyst name in the ticket
4
Pull the initial EDR alert details — detection name, severity, timestamp, process involved, and file path
5
Check the user's recent activity — when did they last log in? Any recent phishing reports? Any MFA anomalies?
2
Volatile Data & Running State
Collect before reboot

Volatile data exists only while the endpoint is running. Capture this first — it is lost on reboot.

1
Review running processes in EDR — look for unknown, unsigned, or unusually named processes
2
Review active network connections — note any connections to unknown external IPs or unusual ports
3
Check parent-child process relationships — legitimate processes spawning cmd.exe, PowerShell, or wscript.exe is a major red flag
4
Review loaded DLLs for any suspicious unsigned or recently created libraries injected into legitimate processes
5
Document all findings with timestamps and screenshots before proceeding
Process spawned from an unexpected parent — e.g. Word.exe → cmd.exe → PowerShell.exe
Process running from temp folders — %TEMP%, %APPDATA%, or C:\Users\Public\
Network connection from a process that should not be making outbound connections (e.g. notepad.exe connecting externally)
Unsigned process running with a name similar to a legitimate system process (e.g. svchost.exe in the wrong location)
3
Windows Event Log Analysis
Key Event IDs

Event logs are stored at C:\Windows\System32\winevt\Logs\ in .evtx format. Focus on Security, System, PowerShell, and Sysmon logs where available.

Authentication & account events

Event IDDescriptionInvestigation significance
4624Successful logonCheck logon type — Type 10 = RDP, Type 3 = network. Note source IP and username.
4625Failed logonMultiple failures before 4624 = brute force. Note failure reason and source IP.
4634LogoffCorrelate with 4624 to establish session duration.
4648Logon using explicit credentialsCredential use with different account — lateral movement or privilege abuse indicator.
4672Special privileges assigned at logonAdmin-level access — always investigate for privileged users.
4720User account createdUnexpected account creation = persistence or backdoor.
4732Member added to local Administrators groupPrivilege escalation — investigate immediately.
4740User account locked outRepeated lockouts indicate brute force or credential stuffing.

Process & execution events

Event IDDescriptionInvestigation significance
4688New process createdNote process name, parent process, and command-line arguments. Requires process auditing enabled.
4689Process terminatedCorrelate with 4688 — very short-lived processes are suspicious.
4698Scheduled task createdCommon persistence mechanism — review task name, trigger, and action carefully.
7045New service installedMalware persistence via service installation. Review service binary path.
7034Service crashed unexpectedlyMay indicate a malicious service failing or being terminated.

Tamper & evasion events

Event IDDescriptionInvestigation significance
1102Security audit log clearedAlmost always malicious — attacker covering tracks. Escalate immediately.
104System log clearedAs above — log clearing is a strong compromise indicator.
4719System audit policy changedAttacker disabling logging. Investigate urgently.
4657Registry value modifiedCheck which key was modified — autorun locations are high priority.

PowerShell events (Microsoft-Windows-PowerShell/Operational)

Event IDDescriptionInvestigation significance

4103Module logging — pipeline execution detailsModule-level activity. Useful for identifying what PowerShell modules were invoked.

4
Persistence Mechanism Review
Check all locations

Malware establishes persistence to survive reboots and credential changes. Check every location below — missing one means reinfection after remediation.

Persistence typeLocation / key to checkWhat to look for
Registry autoruns HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\Run HKLM\SOFTWARE\Microsoft\Windows\CurrentVersion\RunOnce HKCU\SOFTWARE\Microsoft\Windows\CurrentVersion\Run Unknown entries, entries pointing to temp folders, encoded commands
Scheduled tasks C:\Windows\System32\Tasks\ C:\Windows\SysWOW64\Tasks\ Event ID 4698 (task created) Recently created tasks, tasks with encoded actions, tasks running from temp paths
Services HKLM\SYSTEM\CurrentControlSet\Services\ Event ID 7045 (service installed) Unknown service names, services pointing to unusual binary paths, recently installed services
Startup folders C:\Users\[user]\AppData\Roaming\Microsoft\Windows\Start Menu\Programs\Startup\ C:\ProgramData\Microsoft\Windows\Start Menu\Programs\Startup\ Unknown .lnk, .bat, .vbs, or .ps1 files
WMI subscriptions WMI EventFilter + EventConsumer + FilterToConsumerBinding Fileless persistence — hard to detect without WMI query or EDR. Common in advanced attacks.
DLL hijacking / side-loading Application directories alongside legitimate executables Malicious DLL with same name as legitimate DLL loaded by a trusted application
LSASS / credential stores HKLM\SECURITY\Policy\Secrets C:\Windows\NTDS\ (domain controllers only) Evidence of credential dumping — check for lsass.exe memory access in EDR

⚠️ If any persistence mechanism is found — do not remove it yet. Document its exact location, value, and creation timestamp first, then follow EP-WI-03 for approved removal.

5
LOLBin & Living-off-the-Land Activity
Trusted tools abused

LOLBins (Living-off-the-Land Binaries) are legitimate Windows tools abused by attackers to blend in and evade detection. Look for these in process trees, command-line logs, and PowerShell events.

powershell.exe
Download cradles, encoded commands (-EncodedCommand), bypass execution policy, remote execution
High risk
cmd.exe
Spawned by Office apps, scripts, or services — review parent process and command-line arguments
High risk
certutil.exe
File download (-urlcache), base64 decode (-decode) — frequently used for malware staging
High risk
mshta.exe
Executes HTML application files (.hta) — used to run VBScript or JScript payloads
High risk
wscript.exe / cscript.exe
Windows Script Host — executes .vbs and .js files. Often used in phishing payload delivery
High risk
regsvr32.exe
COM script execution via /s /n /u /i — can execute remote scripts without touching disk (Squiblydoo)
High risk
rundll32.exe
Executes DLL exports — used to run malicious DLLs or call specific export functions
High risk
wmic.exe
Remote execution, process creation, persistence via WMI — often used for lateral movement
High risk
bitsadmin.exe
Background Intelligent Transfer — used to download malware payloads and maintain persistence
Medium risk
msiexec.exe
MSI installer — can be used to execute remote MSI packages and DLL payloads
Medium risk
net.exe / net1.exe
User and group enumeration, share mapping — reconnaissance and lateral movement
Medium risk
schtasks.exe
Scheduled task creation and management — persistence and remote execution
Medium risk

⚠️ Context matters — these tools run legitimately every day. The key signals are: unexpected parent process, unusual command-line arguments, execution from a temp/user directory, or execution at an unusual time.

6
File System & Artefact Review
Execution & staging evidence
📁 Prefetch files
C:\Windows\Prefetch\*.pf
Records of executed applications. Proves a binary ran on the system even if it has been deleted.
📋 Recent files (LNK)
C:\Users\[user]\AppData\Roaming\Microsoft\Windows\Recent\
Recently opened files — reveals what the user (or attacker) accessed. Check for unusual file types.
🗂️ Temp directories
C:\Users\[user]\AppData\Local\Temp\ C:\Windows\Temp\
Common malware staging location. Look for executables, scripts, or archives dropped here.
📦 Downloads folder
C:\Users\[user]\Downloads\
Check for suspicious executables, ISO files, archives, or Office documents with macros.
📜 PowerShell history
C:\Users\[user]\AppData\Roaming\Microsoft\Windows\PowerShell\PSReadLine\ConsoleHost_history.txt
All commands typed in PowerShell. Look for download cradles, encoded commands, or credential access attempts.
🖥️ Shimcache / Amcache
HKLM\SYSTEM\CurrentControlSet\Control\Session Manager\AppCompatCache C:\Windows\appcompat\Programs\Amcache.hve
Records of all executables that have run or been present on the system — useful for deleted malware.
🔌 USB device history
HKLM\SYSTEM\CurrentControlSet\Enum\USBSTOR
All USB storage devices ever connected. Useful for data exfiltration investigations.
🌐 Browser artefacts
C:\Users\[user]\AppData\Local\[Browser]\User Data\Default\History
Browser history, downloads, and cached credentials. Relevant for phishing link analysis.
7
Network Activity Correlation
C2 & exfiltration signals
1
Review all outbound connections from the endpoint in the EDR/NDR — note destination IPs, domains, ports, and protocols
2
Enrich suspicious destination IPs and domains — check reputation, registration date, hosting provider, and known malicious associations
3
Look for beaconing patterns — regular periodic connections at fixed intervals strongly suggest C2 communication
4
Check DNS query logs — high-entropy subdomains or unusually long queries may indicate DNS tunnelling
5
Check for connections to known file sharing, cloud storage, or paste sites — potential exfiltration channels
6
Review for internal scanning — connections from this endpoint to many other internal IPs on SMB (445), RDP (3389), or WinRM (5985) ports suggest lateral movement
Beaconing — regular outbound connections every N seconds/minutes to the same destination
HTTPS/TLS traffic to a newly registered domain or IP with no prior history
Large outbound data transfer to an unknown destination — possible exfiltration
Internal port scanning from this endpoint — SMB (445), RDP (3389), WinRM (5985/5986)
DNS queries to high-entropy subdomains — possible DNS tunnelling C2

→ If network indicators are found, invoke NW-WI-01 for deeper network traffic analysis.

8
Identity & Credential Correlation
Lateral movement signals
1
Review the user's sign-in logs in the identity provider — any logins from unusual locations or devices around the same time as the endpoint alert?
2
Check for credential access indicators on the endpoint — lsass.exe memory access, Mimikatz signatures, or SAM database access
3
Check for pass-the-hash or pass-the-ticket indicators — Event ID 4648 (explicit credentials used) or logon Type 3 from the endpoint to other internal systems
4
Review other endpoints for logons using the same username around the same timeframe — lateral movement from this endpoint
5
Check for new local accounts or privilege escalation — Event IDs 4720, 4732

→ If credential compromise is suspected, invoke ID-WI-01 (Review Sign-In Activity) and ID-WI-02 (Reset Compromised Account).

9
Investigation Verdict & Recommendation
Document & decide

Based on all phases, assign a verdict and document the technical rationale clearly. The verdict drives the next action.

✅ Clean

No indicators found across all phases. Document and close with monitoring note.

⚠️ Suspicious

Anomalies present but not confirmed malicious. Continue monitoring or escalate for deeper review.

🟠 Likely Compromised

Multiple indicators present. Invoke EP-PB-01/02, keep isolated, proceed to eradication.

🔴 Confirmed Compromise

Clear evidence of malware execution or attacker activity. Full IR response — EP-PB-02/05.

Next action by verdict:

✅
Clean → Document findings, remove isolation if approved, close ticket
⚠️
Suspicious → Escalate to Tier-2/3 for review, maintain isolation, continue monitoring
🟠
Likely Compromised → Invoke EP-PB-01, proceed to EP-WI-03 (Remove Malware Artefacts)
🔴
Confirmed → Invoke EP-PB-02 or EP-PB-05, notify Tier-3/IR Lead, full eradication and recovery cycle
Common investigation mistakes to avoid
⚠️Rebooting the endpoint before collecting volatile data — destroys running process list, network connections, and in-memory artefacts
⚠️Treating an EDR alert as the full picture — the alert is the starting point, not the conclusion
⚠️Removing malware before fully documenting all persistence locations — misses related mechanisms and causes reinfection
⚠️Skipping identity and network correlation — endpoint compromise almost always has identity and network dimensions
⚠️Assuming a clean EDR scan means the endpoint is clean — fileless malware and LOLBin abuse may not trigger detections
⚠️Notifying the user before isolation is confirmed — alerts the attacker if credentials are already compromised
