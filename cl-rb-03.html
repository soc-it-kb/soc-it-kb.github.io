CL-RB-03: Lock Down Public Cloud Storage

Triage and containment runbook for object storage, data lake, file share, or backup repository public exposure.

Section 1
Purpose

Triage and containment runbook for object storage, data lake, file share, or backup repository public exposure.

This runbook provides repeatable execution steps for analysts and platform operators while preserving evidence and approval traceability.

Section 2
Required Access
Cloud storage administrationStorage access logsData classification or ownership recordNetwork / firewall controlsSOC ticketing system
Section 3
Inputs Required
Storage account/container/bucket/share identifierOwner, classification, and business criticalityCurrent public access, ACL, policy, and sharing stateRelevant access logs and alert timeframe
Section 4
Step-by-Step Procedure
1
Identify storage object
1
Record account/container/bucket/share, resource owner, classification, public access state, network rules, and last modified time.
2
Preserve exposure evidence
1
Capture configuration, access policy, ACLs, sharing links, SAS/pre-signed URLs, and access logs before changing state.
3
Restrict public access
1
Disable anonymous/public access, revoke temporary URLs, limit network access, and apply least privilege to identities.
4
Assess access history
1
Review read/list/download/delete events, source IPs, user agents, principals, and unusual bulk access patterns.
5
Notify owners
1
Inform business data owner, GRC/privacy, and incident lead if sensitive data may have been accessed.
6
Validate closure
1
Confirm no public path remains and baseline policy prevents reintroduction.
Section 5
Expected Outcome

Public or excessive access is removed, exposure evidence is preserved, and data owner/GRC escalation is completed where required.

Section 6
Validation Checks
  • Public access and temporary URLs are removed or approved as exception.
  • Access history was reviewed for read/list/download/delete activity.
  • Data classification and owner are recorded.
  • Preventive baseline or policy action is created if drift caused exposure.
Section 7
Common Errors & Tips
Capture evidence before locking down access; after-state alone is insufficient.
Review list operations as well as downloads.
Confirm all equivalent storage locations in the same workload or resource group.
Section 8
Output Artefacts
Before/after access policy
Storage access log extract
Owner notification
Data classification notes
Closure validation
