Tier-1 Quick Checklist

ID-03: MFA Fatigue / MFA Push Abuse

Use this checklist to triage an MFA push abuse alert. Questions 2 and 3 are critical — if either is YES, treat as account compromise.

🔐 Identity Tier 1 Published 20 Dec 2025
🔔 When to use this checklist
Multiple MFA push alerts received by a user / user self-report / MDR escalation involving repeated MFA prompts.
Multiple MFA push notifications User reports unexpected prompts MDR MFA fatigue escalation

Step 3 — Apply decision logic
🔴
YES to question 2 or 3 — Escalate to Tier-3 immediately. Treat as account compromise. Do not wait.
⚠️
All NO but multiple prompts received — Escalate to Tier-2 for investigation. Multiple prompts alone indicate an active attack attempt.
Step 4 — Mandatory actions
Contact the user immediately to confirm whether they approved any prompt
Preserve MFA logs — capture all prompt timestamps and outcomes
Do not dismiss as user error — always treat as potential attack
Attach all MFA log evidence to the incident ticket
⚠️ Common Tier-1 mistakes to avoid
Waiting for the user to complain — proactively contact the user when multiple prompts are observed
Assuming that MFA denials mean there is no risk — the attacker may still be trying
Not correlating with phishing — MFA fatigue attacks almost always follow a phishing attempt
Delaying escalation to gather more information — escalate immediately and let Tier-2 investigate
💡
Analyst Tip
If a user says "I approved it to make it stop" — assume account compromise immediately and escalate to Tier-3. This is a confirmed MFA fatigue success.
