ID-RB-01: Account Reset

Contain suspected or confirmed account compromise by revoking all active sessions, resetting credentials, and enforcing MFA re-registration.

🔐 Identity threats Tier 1 / 2 Containment
📋 ID: ID-RB-01
·
👤 Author: Paulo Ferreira
·
📅 Published: 22 December 2025
·
🏷️ Category: Identity Threat
·
Active
Section 1
Purpose

Contain suspected or confirmed account compromise by revoking access, resetting credentials, and enforcing MFA.

Section 2
Required Access
🔑 Identity admin privileges 📊 Sign-in log access
Section 3
Inputs Required
👤 Username / UPN 🎫 Incident ID
Section 4
Step-by-Step Procedure
1
Immediate Containment
1
Revoke all active sessions and tokens
Do this before resetting the password — session revocation stops an active attacker immediately
2
Disable the account if active attacker access is confirmed or ongoing
Re-enable after reset and MFA enforcement are complete
2
Credential Reset
1
Reset the account password to a strong temporary password
2
Enforce password change at next sign-in
3
Invalidate all refresh tokens to prevent token-based persistent access
Password reset alone does not revoke existing OAuth tokens — ensure token invalidation is performed separately
3
MFA Enforcement
1
Require MFA re-registration — invalidate existing MFA methods
2
Remove any suspicious or unrecognised registered devices from the account
Ensure the user re-registers MFA on a trusted device only
4
Validation
1
Review recent sign-in logs to confirm no active attacker sessions remain
2
Confirm zero active sessions on the account
5
Documentation
1
Record all actions taken, timestamps, and outcomes in the incident ticket
2
Notify the affected user via an approved out-of-band communication channel
Section 5
Expected Outcome

✅ Account fully secured — attacker access removed
✅ All active sessions revoked and refresh tokens invalidated
✅ MFA enforced and re-registered on a trusted device

Section 6
Validation Checks
  • No active sessions visible in the identity platform
  • MFA enforced and confirmed registered
Section 7
Common Errors & Tips
Always revoke sessions before resetting the password — password reset alone does not terminate active sessions
Check for OAuth abuse separately — password resets do not revoke OAuth tokens granted to third-party apps (see ID-RB-04 / ID-RB-05)
Section 8
Output Artefacts
Sign-in log review
Reset confirmation
