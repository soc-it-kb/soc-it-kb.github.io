IT-GOV-WI-VM-03: Remediation Validation

Validate that remediation actions effectively removed or mitigated vulnerabilities before formal closure.

🔍 Vulnerability Management IT operations Published 13 January 2026
🔧 WI ID: IT-GOV-WI-VM-03
·
👤 Author: Paulo Ferreira
·
📅 Published: 13 January 2026
·
Vulnerability Management Procedure
Required access
Remediation confirmation from IT Operations Vulnerability scanning or validation tools Ticketing system
Step-by-step instructions
1
Confirm Remediation Completion
Confirm patch applied, configuration changed, or control implemented
Obtain confirmation from IT Operations that remediation is complete
2
Re-Scan or Validate
Perform targeted re-scan of affected asset
Apply configuration verification where re-scan is not possible
3
Review Results
Confirm vulnerability is no longer detected
Investigate any partial remediation or persistence of finding
4
Close Finding
Attach validation evidence to ticket
Update ticket status to closed with evidence reference
Expected outcome
Verified remediation confirmed
Evidence attached to ticket
Vulnerability formally closed in tracking system
Common errors & tips
Do not close findings without validation evidence — assumptions are not acceptable
Be aware of scan cache delays — a re-scan may return stale results if run too quickly after patching
Re-open the ticket immediately if the vulnerability reappears on a subsequent scan
