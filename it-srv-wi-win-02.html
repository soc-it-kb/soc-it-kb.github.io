IT-SRV-WI-Win-02: OS Hardening & Service Reduction

Minimise attack surface by disabling unnecessary roles, features, services, and legacy protocols on Windows servers.

🖥️ Server hardening Windows Server IT operations
🔧 WI ID: IT-SRV-WI-Win-02
·
👤 Author: Paulo Ferreira
·
📅 Published: 12 January 2026
·
IT procedure
Prerequisites
Administrative access Server role and function confirmed
Step-by-step instructions
1
Review installed Windows roles and features
2
Remove unused roles and features
3
Disable unnecessary services
4
Disable legacy and insecure protocols (e.g. SMBv1)
5
Restrict script execution to approved use cases
Validation & verification
Only required roles and services enabled
Legacy protocols disabled
Server functionality validated post-hardening
Rollback & troubleshooting
Re-enable required services if business impact occurs
Document and approve all deviations
Common errors & tips
Always validate server role before hardening
Test changes in non-production where possible
