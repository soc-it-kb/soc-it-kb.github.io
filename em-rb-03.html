EM-RB-03: Analyse Attachments

Safely analyse email attachments to determine whether they contain malware, droppers, or credential-harvesting mechanisms — using static and dynamic analysis in an isolated environment.

📧 Email threats Tier 1 / 2 Alert triage
📋 ID: EM-RB-03
·
👤 Author: Paulo Ferreira
·
📅 Published: 22 December 2025
·
🏷️ Category: Email Threat
·
Active
Section 1
Purpose

Safely analyze email attachments to determine whether they contain malware, droppers, or credential-harvesting mechanisms.

Section 2
Required Access
🔬 Secure sandbox 🛡️ Malware analysis tools #️⃣ Hashing utility 🎫 SOC ticketing system
Section 3
Inputs Required
📎 Original attachment file 📧 Source email 🎫 Incident ID
Section 4
Step-by-Step Procedure
1
Secure File Handling
1
Download attachment to an isolated environment only
2
Do not open on production systems under any circumstances
3
Preserve the original filename and metadata
2
Static Analysis
1
Calculate the SHA256 hash of the file
2
Identify file type and extract metadata (magic bytes, embedded objects)
3
Check hash reputation against threat intelligence (VirusTotal, internal TI platform)
4
Note any suspicious attributes — obfuscation, mismatched extensions, embedded URLs
3
Dynamic Analysis
1
Submit file to sandbox environment
2
Observe and record the following behaviours:
  • Process execution and child processes spawned
  • Outbound network connections initiated
  • Files dropped to disk or written to memory
3
Capture all indicators from the sandbox report
4
Identify Malicious Behaviour
1
Look for the following indicators of malicious behaviour:
  • Macro execution or VBA code
  • Script abuse (PowerShell, WScript, CMD)
  • Secondary payload download
2
Identify any persistence attempts made during execution
5
Document Findings
1
Record the verdict — Benign / Suspicious / Malicious
2
Attach the full sandbox report to the incident ticket
3
Record all hashes and IOCs for blocking and threat hunting
Section 5
Expected Outcome

✅ Clear verdict on the attachment (Benign / Suspicious / Malicious)
✅ Indicators available for blocking and threat hunting

Section 6
Validation Checks
  • Hash verified and recorded in the incident ticket
  • No execution outside of the isolated sandbox environment
Section 7
Common Errors & Tips
HTML files often contain credential lures — do not dismiss them as low risk
ISO and IMG files frequently bypass email security filters — always sandbox these
Preserve the original sample — it is evidence and may be required for further analysis
Section 8
Output Artefacts
File hashes (SHA256)
Sandbox analysis report
IOC list
