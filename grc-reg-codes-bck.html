GRC-REG-CODES: Naming Convention Codes Register

The authoritative controlled vocabulary for the asset naming convention defined in GRC-SOP-05. Holds approved agency codes, city/location codes, asset-type codes, and recorded deviations. Codes are added incrementally as operating units onboard.

📌 How this register works. Codes are introduced incrementally — only operating units that have onboarded under GRC-SOP-05 appear in the agency table; only locations actually used by those agencies appear in the city table. Code requests follow the workflow in GRC-SOP-05 §14.1. Once approved, a new code becomes immediately usable in asset names. The seed values below are starter entries authorised at SOP-05 publication; further codes are added as agencies join.
🏢 Agency Codes

Three- to five-letter uppercase codes identifying the operating unit (agency) that owns an asset. Used as the first segment of the asset name. Special value CENTRAL is reserved for cross-agency assets provided centrally and consumed by multiple operating units (see GRC-SOP-05 §7.3).

📨 Code Request Workflow

When a new code is needed (new agency onboarding, new city, new asset type), submit a request following the steps below. Approved codes appear in the relevant table within five working days.

1
Identify the gap — Determine which table is affected (Agency / City / Asset Type) and confirm the code does not already exist (search this register first).
2
Propose a code — Three letters for cities and asset types; three to five letters for agencies. Uppercase A–Z only. Where IATA airport codes exist for cities, prefer those.
3
Submit to GRC — Email the GRC team with: proposed code, full name, justification, expected scope, sponsor (LCSO or Local IT Lead).
4
Review & consultation — Head of GRC reviews for collisions and intuitive uniqueness. Asset-type codes consulted with Architecture; cross-agency codes consulted with CIO.
5
Approval & publication — Once approved, code is added to this register with date and owner. Submitter is notified. Code is then available for use.
