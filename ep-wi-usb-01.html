EP-WI-USB-01: Handle USB Mass Storage with Suspected Malicious Files

Consistent, safe, and forensically sound procedure for handling USB devices suspected of containing malicious content — preventing execution, preserving evidence, and supporting investigations.

💻 Endpoint threats Tier 1 / 2 Requires approval Forensic handling
🔧 WI ID: EP-WI-USB-01

Section 1
Purpose

Provide a consistent, safe, and forensically sound procedure for handling USB mass storage devices suspected of containing malicious content — preventing execution, preserving evidence, and supporting investigations involving malware delivery or insider threats.

📌 Scope boundary: This WI does not authorise USB usage. It defines how to safely handle and analyse USB media once discovered. Any analysis requires Tier-2 approval.

Section 2
When to Use
User reports suspicious USB device Malware detected after USB insertion USB found unattended or unexpectedly connected EP-PB-01 Suspected Compromise EP-PB-02 Malware Execution Insider threat or forensic investigation MDR / EDR removable media alert

RoleResponsibility

Section 4
Inputs Required
🖥️ Endpoint hostname / device ID 👤 User involved (if applicable) 🎫 Incident reference ID 📦 Physical access confirmed ✅ Approval to proceed
Section 5
Pre-Checks & Safety Notes
🚫 Critical safety rules — no exceptions
Never open files directly from the USB on any production endpoint
Never allow autorun or auto-mount execution
Do not reuse the USB on multiple systems
Assume the USB is hostile until proven otherwise

⚠️ If the USB is currently connected to a production endpoint and malware is suspected — pause and escalate to Tier-2 immediately before removing the device.

Section 6
Step-by-Step Actions
1
Secure the Endpoint
1
If USB is currently connected — do not interact with files. Disconnect endpoint from network if instructed. Follow EP-WI-02 if malware execution is suspected or EDR alerts are present. Document exact state before USB removal.
2
Safely Remove & Label the USB Device
1
Safely eject the USB without browsing contents. Physically label the device with: Incident ID, date/time, and finder/user.
2
Store in secure SOC or IT storage. Do not reconnect to any standard endpoint.
3
Preserve Evidence
1
Record and preserve:
  • USB make, model, and capacity
  • Serial number (if visible)
  • Where and how it was discovered
  • Endpoint and user context
Take photos if physical condition is relevant to the investigation.
4
Decide Analysis Path (Tier-2 Approval Required)
Analysis pathWhen to use
No analysis — retain as evidence onlyLegal / HR involvement or low-priority incident
Controlled malware analysisMalware suspected — isolated analysis system only
External forensic analysisHigh-severity or insider threat investigation

⚠️ Tier-1 must not proceed to analysis without Tier-2 approval. Do not select the analysis path yourself.

5
Controlled Analysis (If Approved)
1
Use a dedicated, isolated analysis system with no network access and no production credentials. Disable autorun and script execution. Mount USB in read-only mode where possible.
2
Identify file names and extensions, creation/modification timestamps, and hidden or executable files.
Do not double-click or execute any files — ever.
6
Malware Identification (If In Scope)
1
Extract file hashes only — no execution. Submit hashes to approved malware analysis processes. Correlate with endpoint detections, network indicators, and threat intelligence.
7
Correlate with Endpoint Activity
1
Check whether the USB was previously mounted, files were copied to the endpoint, or processes executed from removable media paths.
If endpoint execution occurred — escalate to EP-PB-01 / EP-PB-02 immediately.
8
Containment & Cleanup (If Required)
1
If malicious files were copied or executed — follow EP-WI-03 (Remove Malware Artefacts) and EP-WI-04 (Validate Persistence Removal). Do not return the USB to service.
9
Disposal or Retention

🔒 Retain as evidence

Ongoing investigation — keep in secure storage with chain of custody

🔥 Destroy securely

Per policy — document destruction method and date

⚖️ Hand to Legal / HR

Insider threat or disciplinary matter — transfer with chain of custody record

10
Document Actions
1
Update incident record with: discovery details, handling steps, analysis outcome (if any), endpoint impact assessment, and final disposition. Attach photos, logs, and analysis notes.
Section 7
Expected Outcome

✅ USB-borne threat risk contained
✅ No malicious files executed unintentionally
✅ Evidence preserved with chain of custody maintained
✅ Endpoint impact fully assessed and documented

Section 8
Common Mistakes to Avoid
Opening files to "see what they are" — this executes malware and destroys forensic integrity
Using personal or production machines for analysis — always use a dedicated isolated system
Letting the user keep or reuse a suspicious USB device
Skipping endpoint correlation — files may have already been executed before the USB was reported
Losing physical chain of custody — label and document every transfer of the device
Section 9
Escalation Conditions
Executables are present on the USB
Malware execution is suspected or confirmed on the connected endpoint
USB was connected to multiple endpoints
USB targets privileged or sensitive systems
Legal, HR, or insider threat implications are present
