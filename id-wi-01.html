ID-WI-01: Review Sign-In Activity

Tool-agnostic procedure for reviewing user sign-in logs to identify account compromise, anomalous authentication, MFA abuse, and suspicious access patterns — investigation only, no containment actions.

Section 1
Purpose

Provide a consistent, tool-agnostic procedure for reviewing user sign-in activity to identify account compromise, anomalous authentication behaviour, MFA abuse or bypass, and suspicious locations, devices, or access patterns.

📌 Scope boundary: This WI supports initial triage and evidence collection only. No credential resets, session revocations, or containment actions are performed here.

Section 6
Step-by-Step Actions
1
Identify Relevant Sign-In Records
1
Locate authentication logs for the target user and filter by user identifier and the relevant timeframe
2
Ensure you include both successful and failed sign-ins, and MFA challenges if logged
Focus primarily on successful sign-ins — failed attempts show the attacker was trying; successful ones show they got in.
2
Review Authentication Outcomes
1
For each sign-in event record: timestamp, result, authentication method, and failure reason if applicable. Flag:
  • Multiple failures followed by success
  • Successful sign-in after repeated MFA denials
  • Successful sign-in without MFA where MFA is expected
3
Review Source Location & Network Indicators
1
For each event, record source IP, geographic location, and ASN or network provider. Flag:
  • Location unusual for the user
  • Multiple distant locations within a short timeframe (impossible travel)
  • Source is anonymised — TOR, VPN, hosting provider, or proxy
4
Review Device & Client Context
1
Record device ID/name, compliance/trust state, and client application. Flag:
  • New or unknown device
  • Legacy or weak authentication protocols (IMAP, POP, Basic Auth)
  • Client application that doesn't match the user's normal behaviour
5
Review MFA Behaviour
1
If MFA is enabled, record number of prompts, approval vs denial, and MFA method. Flag:
  • High volume of MFA prompts in short period
  • MFA approved from a suspicious or unexpected location
  • MFA absent where expected — possible token theft (see ID-WI — Token Theft)
6
Correlate with Other Signals
1
Check for correlation with phishing reports (EM-PB-xx), OAuth consent events (ID-PB-04), endpoint alerts (EP-PB-xx), and network anomalies (NW-PB-xx). Record any correlation references.
7
Summarise Findings & Assign Confidence

✅ Benign

All activity consistent with user pattern — no anomalies

⚠️ Suspicious

Anomalies present — escalate to Tier-2

🔴 Likely Compromised

Multiple indicators — invoke relevant playbook

📌 Document the specific indicators, timestamps, and source values that support the classification — do not assign confidence without technical rationale.

Section 7
Expected Outcome

✅ Clear picture of the user's recent authentication activity
✅ Evidence to support escalation or closure with confidence classification
✅ No containment actions performed — investigation only

Section 8
Common Mistakes to Avoid
Focusing only on failed sign-ins — successful sign-ins are where the compromise shows
Assuming VPN usage is benign without confirmation from the user or IT
Overlooking legacy authentication protocol usage — IMAP/POP/Basic Auth bypass MFA entirely
Not correlating sign-in anomalies with phishing or endpoint events
Section 9
Escalation Conditions
Successful sign-in from a suspicious or impossible location
MFA abuse indicators present — fatigue, bypass, or missing challenge
Impossible travel pattern detected
Sign-in activity correlates with phishing or endpoint compromise
Privileged or admin account involved
