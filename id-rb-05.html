ID-RB-05: Revoke OAuth Consent & Invalidate Tokens

Safely revoke OAuth consent, invalidate tokens, and remove persistent access granted to malicious or unauthorised applications — ensuring attacker access is fully and verifiably terminated.

Section 1
Purpose

Safely revoke OAuth consent, invalidate tokens, and remove persistent access granted to malicious or unauthorised applications, ensuring attacker access is fully terminated.

This runbook executes containment and eradication for OAuth abuse discovered during ID-PB-04 investigations.

Section 2
Required Access
🔑 Entra ID application admin access ⚙️ Enterprise application management 🔒 Token revocation permissions 🎫 SOC ticketing system
Section 3
Inputs Required
🆔 Application ID or name 👤 Affected user(s) 🎫 Incident reference ID
Section 4
Step-by-Step Procedure
1
Revoke User Consent
1
Navigate to Enterprise Applications in the Entra ID admin portal
2
Locate the identified malicious or unauthorised application
3
Remove the following from the application:
  • User consent grants
  • Delegated permissions
4
Confirm consent removal is complete and recorded in the audit log
2
Remove Application Access

Choose the appropriate scope based on the extent of abuse:

👤 User-scoped abuse

  • Remove user assignment to the app only

🏢 Tenant-wide abuse

  • Disable the application entirely
  • Remove all user assignments
1
Apply the appropriate scope and document the extent of removal
3
Invalidate Tokens
1
Revoke all refresh tokens for the affected user(s)
This is the critical step — removing permissions without revoking tokens leaves the attacker with active access
2
Force token invalidation across all active OAuth sessions
3
Confirm no active OAuth sessions remain for the affected user and application
4
Validate Access Removal
1
Review audit logs to confirm successful revocation
2
Monitor for the following post-revocation indicators:
  • Continued access attempts from the app
  • Token usage without a corresponding interactive sign-in
3
Confirm access termination — re-test if necessary
5
Coordinate Account Containment (If Required)
1
If OAuth abuse followed a credential compromise, invoke ID-RB-01 to reset the account and enforce MFA re-registration
OAuth abuse and credential compromise frequently occur together — treat both in parallel
6
Document Actions
1
Record the following in the incident ticket:
  • Applications revoked and scope (user vs tenant)
  • Users affected
  • Tokens invalidated and timestamps
2
Attach screenshots or exported audit log entries to the ticket
3
Update the incident status
Section 5
Expected Outcome

✅ OAuth app access fully and verifiably revoked
✅ All tokens invalidated — no residual access without re-authentication
✅ Audit logs confirm revocation

Section 6
Validation Checks
  • App no longer authorised in Enterprise Applications
  • No further token-based access observed
  • Audit logs confirm revocation with timestamps
Section 7
Common Errors & Tips
Removing the app registration alone does not revoke access — you must explicitly invalidate tokens
Always revoke tokens, not just permissions — an attacker with a valid refresh token will retain access even after consent removal
Check whether multiple users consented to the same app — tenant-wide review may be needed
Re-test after revocation by attempting to access a protected resource using the app — confirm it fails
Section 8
Output Artefacts
Revocation confirmation
Audit log entries
Updated incident timeline
