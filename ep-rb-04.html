EP-RB-04: Contain Lateral Movement

Contain and disrupt lateral movement by cutting active access paths, blocking credential reuse, and preventing further endpoint-to-endpoint spread across the environment.

💻 Endpoint threats Tier 2 / 3 Containment

Section 1
Purpose

Contain and disrupt lateral movement by:

  • Cutting active access paths between endpoints
  • Blocking credential reuse across systems
  • Preventing further endpoint-to-endpoint spread

This runbook supports EP-PB-03 (Lateral Movement) and EP-PB-05 (Ransomware Activity).

⚡
Lateral movement frequently precedes ransomware deployment. Speed of containment is critical — every uncontained host is a potential encryption target.
Section 2
Required Access
🛡️ EDR platform 🔑 Identity platform (session control) 🌐 Network / firewall coordination channel 🎫 SOC ticketing system
Section 3
Inputs Required
🖥️ Source endpoint(s) 🖥️ Destination endpoint(s) 👤 Compromised account(s) 🎫 Incident reference ID
Section 4
Step-by-Step Procedure
1
Contain Source Endpoint
1
Confirm the source endpoint from investigation telemetry
2
Isolate the source endpoint if not already isolated — invoke EP-RB-02
3
Validate isolation success before proceeding to destination containment
2
Contain Destination Endpoints
1
Identify all destination hosts that were accessed by the attacker
2
Investigate each destination endpoint for signs of execution or persistence (invoke EP-RB-01)
3
Isolate destination endpoints where compromise is suspected or confirmed
3
Contain Credentials
1
Identify all accounts used for lateral movement across affected hosts
2
Invoke ID-RB-01 for each compromised account to:
  • Revoke all active sessions and tokens
  • Reset credentials
3
Confirm no active sessions remain for any of the identified accounts
4
Block Movement Paths
1
Coordinate with the network team to apply the following temporary controls:
  • Block SMB and RDP where not operationally required
  • Restrict east-west traffic between affected network segments
All network controls must be coordinated with IT before applying to avoid unintended service disruption
2
Document all temporary controls applied — include scope, timestamp, and expected duration
5
Validate Containment
1
Monitor EDR telemetry for:
  • No further remote execution between hosts
  • No reuse of the compromised credentials
2
Confirm containment is effective — no new lateral movement paths observed
6
Document Actions
1
Record the following in the incident ticket:
  • All endpoints isolated (source and destinations)
  • All credentials reset
  • All network controls applied
2
Attach EDR containment evidence and network control records to the ticket
Section 5
Expected Outcome

✅ Lateral movement fully halted — no further east-west spread
✅ All compromised access paths closed
✅ Environment stabilised and movement paths blocked

Section 6
Validation Checks
  • No new east-west connections observed in EDR or network telemetry
  • No reuse of compromised credentials detected
Section 7
Common Errors & Tips
Resetting credentials before isolating the source endpoint may alert the attacker and trigger destructive or data-destruction actions — always contain source first
Always isolate the source endpoint before addressing destinations — stopping the origin prevents further spread during investigation
Lateral movement is frequently a precursor to ransomware deployment — treat speed of containment as critical regardless of current severity classification
Section 8
Output Artefacts
Endpoint containment log
Credential reset confirmation
Network control record
