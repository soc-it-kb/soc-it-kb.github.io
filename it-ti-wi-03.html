IT-TI-WI-03: IOC Actioning & Deployment

Step-by-step instructions for deploying validated Indicators of Compromise to security controls — including firewall, DNS, EDR, email gateway, and SIEM — and confirming deployment success.

Prerequisites
Validated IOC with confidence score — from IT-TI-WI-02 SOC Lead approval for blocking controls Access to relevant control platform (firewall, DNS, EDR, SIEM, email gateway)
Step-by-step instructions
1
Confirm IOC confidence score and deployment decision — High confidence deploys to blocking, Medium to detection/alerting only, Low to logging only
2
Select the appropriate control(s) based on IOC type — see deployment target table below
3
For blocking deployments — obtain SOC Lead approval before proceeding
4
Follow change management process for production control changes where required
5
Deploy the IOC to each selected control. For EDR custom IOC lists — add hash, verify sync to all agents. For DNS blocking — add domain to block list, test resolution returns NXDOMAIN. For firewall — add IP block rule, confirm rule ordering and logging
6
Test the deployment — attempt to resolve the blocked domain or connect to the blocked IP from a test endpoint to confirm the block is active
7
Monitor the deployed IOC for 24–48 hours after deployment — check for false positives or unexpected service disruption
8
Document deployment: control(s) deployed to, timestamp, approver, test results, and monitoring outcome
9
Set a calendar reminder for IOC expiry review — remove or renew at expiry per IT-TI-SOP-02

Validation & verification
Deployment tested and confirmed active
No false positives detected in first 24–48 hours
Deployment documented in IOC register with timestamps
Expiry review reminder set
Common errors & tips
Always test a block after deploying — a misconfigured rule that silently fails gives you false confidence
For IP blocks, check whether the IP is currently in use by a CDN or shared hosting before deploying — use tools like Shodan or a reverse DNS lookup
Deploy to detection first on Medium confidence IOCs — if you see hits, consider escalating to blocking
Document your rollback procedure before deploying — if the block causes service disruption you need to be able to remove it quickly
