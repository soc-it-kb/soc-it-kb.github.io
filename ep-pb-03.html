EP-PB-03: Lateral Movement Activity

Response framework for detecting and containing lateral movement — where an attacker moves from one compromised endpoint to others. Focus is on containment and scope control, as lateral movement significantly increases blast radius.

Section 1
Purpose

Provide a standardised decision and response framework for detecting, investigating, and responding to lateral movement activity, where an attacker attempts to move from one compromised endpoint to others within the environment.

This playbook focuses on containment and scope control, as lateral movement significantly increases blast radius and business impact.

Section 2
Scope

✅ In scope

  • East-west movement between endpoints or servers
  • Use of stolen credentials or tokens for remote access
  • Remote execution mechanisms (SMB, RDP, WinRM, WMI, PsExec)
  • Lateral movement detected by EDR, identity telemetry, or MDR
Common LM techniques
SMB RDP WinRM WMI PsExec Pass-the-Hash

❌ Out of scope

  • Initial endpoint compromise → EP-PB-01 / EP-PB-02
  • Privileged account abuse → ID-PB-05
  • Network-only scanning without endpoint access → Network playbooks
Section 3
Trigger Conditions
  • EDR detects remote execution or suspicious authentication between hosts
  • MDR escalates suspected lateral movement
  • SOC observes credential reuse across multiple endpoints
  • EP-PB-01 or EP-PB-02 investigation identifies spread attempts
  • Identity telemetry shows rapid sign-ins across multiple hosts
Section 4
Initial Validation (Decision Logic)

Confirm the activity represents lateral movement, not legitimate administration:

  • Are source and destination hosts unrelated operationally?
  • Is the user or service account expected to access the destination host?
  • Is the access outside approved admin tooling or change windows?
  • Are credentials being reused across multiple systems?

If YES — legitimate admin activity confirmed → Document and close as Benign Admin Activity
If NO or UNCLEAR → Proceed with lateral movement assessment (Section 5)

Section 5
Lateral Movement Assessment

Assess the technique and scope. Presence of any indicator = Confirmed Lateral Movement Attempt:

SMB or admin share access between unrelated hosts
Remote process creation (PsExec, WMI, WinRM)
RDP or remote shell from a compromised host
Credential reuse observed across multiple systems
Discovery or scanning activity observed from the source
Section 6
Severity & Escalation Criteria

Escalate to High Severity if any of the following apply:

🔴 Multiple endpoints accessed or actively targeted
🔴 Privileged credentials used or exposed during movement
🔴 Server or critical system involved
🔴 Lateral movement follows confirmed malware execution
🔴 MDR confirms active attacker control

Section 7
Response Strategy

Execute the following actions immediately:

Source endpoint containmentCritical
Destination endpoint assessmentCritical
Credential exposure assessmentCritical
Endpoint and identity telemetry reviewCritical
IOC generation and blockingCritical
Mandatory scope expansionCritical

⚠️ If domain controllers, file servers, or management servers are involved → escalate to ID-PB-05 (Privileged Account Compromise) and activate Incident Command immediately

Section 9
Containment Decision Logic

Containment must prioritise source control — coordinated to avoid alerting the attacker prematurely:

🔴 Immediate isolation of source

  • Active lateral movement detected right now
  • Credentials actively being reused across systems
  • Multiple destination hosts already targeted

🟡 Selective destination isolation

  • Execution confirmed on destination endpoints
  • Persistence suspected on destination hosts
Section 10
Scope Expansion Logic

Scope expansion is mandatory for all EP-PB-03 incidents:

Identification actions

  • Identify all endpoints accessed by the compromised account
  • Search for similar lateral movement techniques across EDR
  • Identify any newly compromised endpoints

Correlation actions

  • Correlate network connections between hosts
  • Review authentication logs for credential reuse patterns
  • Cross-reference with open MDR incidents
Section 11
Communication & Coordination
  • Restrict all communications to need-to-know personnel
  • Coordinate endpoint isolation carefully with IT operations
  • Do not notify affected users until containment is fully executed
  • Align actions with MDR and Incident Command workflows
Section 12
Closure Criteria
  • All lateral movement paths identified and fully contained
  • All compromised endpoints isolated, cleaned, or rebuilt
  • All exposed credentials reset
  • No further east-west movement observed
  • Monitoring confirms environment stability
Section 13
Evidence to Capture
Source and destination hostnames
Remote access methods used
Authentication logs and credential usage
Process creation events
Network connections between endpoints
Full timeline of lateral movement
Section 14
Post-Incident Actions
  • Review network segmentation and firewall rules
  • Assess and tighten the administrative access model
  • Enforce least-privilege and credential hygiene
  • Tune EDR and identity detections for lateral movement techniques
  • Feed indicators into threat hunting and SIEM
  • Conduct a lessons-learned session if blast radius was significant
