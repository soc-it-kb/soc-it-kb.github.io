NW-RB-03: Block Internal Scanning / East-West Traffic

Controlled, auditable procedure to contain internal reconnaissance and east-west scanning — restricting network paths used for discovery and preventing progression to lateral movement.

🌐 Network threats Tier 2 East-west containment

Section 1
Purpose

Provide a controlled, auditable procedure to contain and disrupt internal reconnaissance and east-west scanning by restricting network paths used for discovery and enumeration.

This runbook is executed under NW-PB-03 (Internal Scanning), EP-PB-03 (Lateral Movement), and EP-PB-05 (Ransomware Activity — prevent spread).

Section 2
Required Access
🔥 Internal firewall / segmentation controls 📡 NDR visibility (east-west traffic) 🗺️ Network topology documentation 🎫 SOC ticketing system
Section 3
Inputs Required
📤 Source IP / hostname 📥 Destination subnet(s) or hosts 🔢 Protocols and ports observed 🎫 Incident reference ID ⏱️ Containment scope (temporary / permanent)
Section 4
Step-by-Step Procedure
1
Validate Reconnaissance Source
1
Confirm the source system type — user endpoint, server, or management appliance
2
Validate the activity is NOT authorised scanning:
  • Approved vulnerability scanning
  • Patch management or inventory tooling
3
If legitimacy cannot be confirmed — proceed with containment
2
Identify Affected Network Scope
1
Identify source and destination network segments
2
Determine whether scanning is lateral (same segment) or routed (cross-segment)
3
Record all impacted VLANs, subnets, and security zones
3
Determine Containment Strategy
Select the appropriate approach based on risk:
ScenarioRecommended action
User endpoint scanningBlock east-west traffic from source endpoint
Server scanning peersRestrict server-to-server access
Multiple segments targetedSegment-level deny rule
Critical system involvedImmediate isolation + escalation to Incident Command
4
Implement East-West Blocking Controls
1
Apply one or more of the following (as approved):
  • Block source → destination subnet traffic
  • Deny common recon ports (SMB, RPC, LDAP)
  • Restrict management protocols (RDP, WinRM, SSH)
  • Apply temporary segmentation rules
2
Record for each control applied:
  • Control type and scope (source / destination / protocol)
  • Rule ID or change reference
  • Timestamp and operator
5
Coordinate with Endpoint Containment
1
If the source is an endpoint — confirm EP-RB-02 (Isolate Endpoint) status
2
Avoid duplicate isolation actions — coordinate with the endpoint team before applying both network and EDR controls
6
Validate Containment Effectiveness
1
Monitor NDR and firewall telemetry for:
  • No further scanning attempts from the source
  • Block events actively logged
  • No new hosts being probed
2
If scanning persists — expand block coverage and escalate to Incident Command
7
Monitor for Follow-On Behaviour
1
Post-containment, actively monitor for:
  • Credential reuse attempts
  • C2 communication indicators
  • Lateral movement via alternate protocols
2
Escalate to EP-PB-03 if execution is detected, or NW-PB-02 if C2 indicators emerge
8
Document Actions
1
Update SOC ticket with source, destinations, controls applied, and validation results
2
Attach firewall / NDR logs and change references. Record rollback plan for any temporary controls.
Section 5
Expected Outcome

✅ Internal reconnaissance activity halted
✅ Attacker network visibility reduced
✅ Progression to lateral movement prevented

Section 6
Validation Checks
  • No further east-west scanning observed in NDR telemetry
  • Block rules actively enforced
  • Endpoint and identity telemetry stabilised
Section 7
Common Errors & Tips
Blocking too broadly can disrupt operations — scope the block to the specific source and destination, not the entire network
Reconnaissance frequently precedes ransomware — act decisively even when the threat seems contained to scanning only
Always confirm whether the scanner is authorised before blocking — legitimate tooling looks identical to attacker enumeration
Temporary blocks must have an explicit review and expiry timestamp — document this at time of creation
Section 8
Output Artefacts
Containment confirmation
Rule / change references
Validation evidence
Updated incident timeline
