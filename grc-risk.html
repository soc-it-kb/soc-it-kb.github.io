Supplier Security Assessment
Assessment Types

The supplier security assessment programme covers three categories of supplier — each with a dedicated policy, procedure (SOP), questionnaire, scored assessment record, and reviewer work instructions.

☁️ SaaS / Cloud Suppliers

Cloud-hosted applications and platforms. Assessment focuses on data residency, access controls, shared responsibility model, and cloud security posture.

🖥️ On-Prem Application Support

Vendors and support partners with access to on-premises systems. Assessment focuses on remote access controls, credential management, and support scope.

🏢 General Suppliers

All other suppliers with access to systems, data, or facilities. Baseline assessment covering fundamental security hygiene and contractual obligations.

Process overview
Supplier Assessment Lifecycle
1
Identify supplier category — SaaS/Cloud, On-Prem Support, or General. Select the appropriate policy, SOP, and questionnaire.
2
Issue questionnaire — Send the appropriate assessment questionnaire (GRC-FRM) to the supplier for completion.
3
Review & score — Reviewer uses the applicable Work Instruction (GRC-WI) and Scored Assessment Record (GRC-FRM) to evaluate supplier responses.
4
Risk decision — Based on scoring outcome: approve, approve with conditions, request remediation, or escalate for risk acceptance.
5
Document & track — Record outcome in the Risk Acceptance Log. Schedule re-assessment date. Communicate result to relevant stakeholders.
