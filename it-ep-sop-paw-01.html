IT-EP-SOP-PAW-01: Privileged Access Workstation Addendum — Windows

Security requirements for Privileged Access Workstations (PAWs) — hardened, dedicated devices used exclusively for Tier-0 / Tier-1 administrative activities.

Section 1
Purpose

This addendum defines the recommended security requirements for Privileged Access Workstations (PAWs). PAWs are hardened, dedicated devices used exclusively for Tier-0 / Tier-1 administrative activities. Their purpose is to protect high-value credentials, prevent credential theft, and reduce the risk of domain-wide compromise.

⚠️ This addendum supplements — does not replace — IT-EP-SOP-Win-01. PAWs must first meet the full Windows Endpoint Baseline, then apply all additional controls defined here.

Section 2
Threat Model — Why PAWs Exist

PAWs mitigate credential harvesting (phishing, malware, LSASS dumping), token theft and replay, lateral movement from user endpoints, and privilege escalation via compromised admin credentials. The PAW model assumes standard user devices are not trusted for privileged operations.

Section 3
PAW Design Principles

🎯 Dedicated use only

PAWs are for admin tasks only — never for email, browsing, or general productivity

🔐 Credential protection first

LSASS, Credential Guard, and no credential reuse are non-negotiable

🛡️ Reduced attack surface

Minimal software, services, and network access

👁️ Visibility by default

Enhanced SOC monitoring on all PAWs

Section 4
PAW Security Requirements

🔐 Account Separation (Non-Negotiable)

  • Separate standard user and privileged admin accounts
  • Privileged accounts never used on non-PAW devices
  • PAWs not used with standard user accounts

🛡️ Authentication & Credential Protection

  • MFA enforced
  • LSASS protected process enabled
  • Credential Guard enabled where supported
  • NTLM restricted or disabled where feasible

🚫 Software & Usage Restrictions

  • No email clients
  • No general web browsing
  • No productivity or collaboration tools
  • No removable media unless explicitly approved

🌐 Network Access Restrictions

  • Access only to Domain Controllers, management interfaces, and admin portals
  • No direct internet access unless explicitly approved
  • Network segmentation enforced where possible
Section 5
Roles & Governance

Owner: Cyber Security / Identity Services  ·  Review Cycle: Annual or post-incident  ·  Exceptions: Require Cyber Security Governance Board approval — time-bound with compensating controls

Non-compliance with PAW requirements may result in revocation of privileged access.
