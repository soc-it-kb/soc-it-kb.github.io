EP-RB-02: Isolate Endpoint

Safely and rapidly isolate a compromised endpoint from the network to prevent lateral movement, C2 communication, data exfiltration, and ransomware spread.

💻 Endpoint threats Tier 1 / 2 Containment
📋 ID: EP-RB-02
·
👤 Author: Paulo Ferreira
·
📅 Published: 23 December 2025
·
🏷️ Category: Endpoint Threat
·
Active
Section 1
Purpose

Safely and rapidly isolate a compromised endpoint from the network to prevent:

  • Lateral movement to other hosts
  • Command-and-control (C2) communication
  • Data exfiltration
  • Ransomware spread across the environment

This runbook supports EP-PB-01 through EP-PB-05.

Section 2
Required Access
🛡️ EDR console with isolation privileges 📦 Asset inventory visibility 🎫 SOC ticketing system
Section 3
Inputs Required
🖥️ Endpoint hostname or device ID 🎫 Incident reference ID 📝 Justification for isolation
Section 4
Step-by-Step Procedure
1
Validate Endpoint Identity
1
Locate the endpoint in the EDR console using hostname or device ID
2
Confirm all of the following match the incident scope:
  • Hostname
  • Currently logged-in user
  • IP address
3
Verify the endpoint matches the incident scope before proceeding
Isolating the wrong endpoint causes business disruption — double-check hostname carefully
2
Initiate Isolation
Choose the appropriate isolation type:

🔴 Full isolation

EDR-controlled network only — all other traffic blocked. Use for confirmed active compromise or ransomware.

🟡 Restricted isolation

Allows specific traffic (e.g. management). Use when some connectivity must be maintained. Check if supported by your EDR.

1
Select the appropriate isolation type and execute the isolation command in the EDR console
2
Record the exact timestamp of isolation execution
3
Validate Isolation
1
Confirm the endpoint status shows as Isolated in the EDR console
2
Verify:
  • No new outbound network connections from the host
  • EDR heartbeat / management channel still active
3
Monitor for isolation failure alerts — re-execute if isolation did not apply successfully
4
Coordinate with IT (If Required)
1
Notify IT operations of the isolation and expected duration
2
Confirm no critical business service dependency is impacted by the isolation
3
Document all coordination in the incident ticket
5
Document Actions
1
Record the following in the incident ticket:
  • Isolation type applied (full or restricted)
  • Exact timestamp of isolation
  • Operator who executed the action
2
Attach confirmation screenshot from the EDR console to the ticket
Section 5
Expected Outcome

✅ Endpoint fully isolated from the network
✅ No lateral movement or external communication possible
✅ Endpoint remains manageable via EDR management channel

🛑
Do not remove isolation until remediation (EP-RB-03) is fully complete and approved.
Section 6
Validation Checks
  • EDR console reports isolation status as active
  • No new network telemetry observed from the isolated host
Section 7
Common Errors & Tips
Always isolate before malware removal in ransomware scenarios — removing malware on an active network can trigger data destruction or spread
Isolation may disrupt users on VPN — validate the hostname carefully before executing to avoid isolating the wrong device
Do not remove isolation until remediation is fully complete and signed off — partial remediation followed by de-isolation can allow the attacker to re-establish access
Section 8
Output Artefacts
Isolation confirmation (EDR screenshot)
Timeline entry with timestamp
