EM-PB-03: Malicious URL / Link Incident Response

Guidance for SOC analysts handling malicious URLs delivered via email — covering credential harvesting attempts, redirectors, and drive-by download scenarios.

📧 Email threats Tier 1 / 2 / 3 MDR integrated

👤 Author: Paulo Ferreira

Section 1
Purpose

Guide SOC analysts in handling malicious URLs delivered via email, including credential harvesting and drive-by download attempts.

Section 2
Scope
  • URLs embedded in emails
  • Shortened or obfuscated links
  • QR-code-based links
Section 3
Trigger Conditions
  • URL flagged by email security gateway
  • User reports a suspicious link
  • MDR escalation involving a suspicious URL
Section 4
Initial Validation

Confirm the following before proceeding to classification:

  • Is the sender external?
  • Is the URL obfuscated or shortened?
  • Is there evidence of brand impersonation?

If NO to all → Close or downgrade
If YES to any → Proceed to classification (Section 5)

Section 5
Classification Logic

Classify the URL into one of the following categories:

✅ BenignNo malicious behaviour detected
↪️ RedirectorLink leads to a malicious destination site
🔑 Credential HarvestingFake login page designed to capture credentials
💀 Malware HostingURL delivers a payload (drive-by download)
Section 6
User Interaction Assessment

Determine the extent of user interaction with the link:

  • Was the link clicked?
  • Were credentials submitted on the destination page?
  • Was an MFA prompt triggered following the visit?

If credentials submitted or MFA triggered → escalate severity immediately and invoke ID-RB-01 for account containment

Section 7
Escalation Criteria

Escalate to High Severity if any of the following apply:

🔴 Credentials submitted by the user
🔴 Privileged or VIP account involved
🔴 Multiple users impacted by the same URL campaign

Section 8
Response Strategy

Execute the following actions based on classification and user impact:

Detonate URL in sandbox environmentRequired
Expand scope — search for other recipients of same URLRequired
Contain user account (if interaction confirmed)Conditional
Remove email artefacts from mailboxesConditional
Block URL / domainConditional
Section 9
Runbook Invocation Matrix

Invoke the relevant execution runbooks based on the scenario:

URL analysis EM-RB-02 › Email removal from mailboxes EM-RB-04 › Block URL / domain EM-RB-05 › Account reset / containment ID-RB-01 ›
Section 10
Closure Criteria

An incident may be closed when all of the following are confirmed:

  • URL verdict documented and IOCs recorded
  • Affected user(s) remediated
  • Detection rules tuned to prevent recurrence
Section 11
Evidence to Capture
File / URL hashes
Sandbox detonation report
Email artefacts
User interactions report
