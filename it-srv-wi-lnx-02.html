IT-SRV-WI-LNX-02: Identity, SSH & Privileged Access Hardening

Enforce strong identity controls, secure remote access, and least-privilege administration on Linux servers.

🖥️ Server hardening Linux Server IT operations
🔧 WI ID: IT-SRV-WI-LNX-02
·
👤 Author: Paulo Ferreira
·
📅 Published: 16 January 2026
·
IT procedure
Prerequisites
SSH access to server Central identity integration where applicable
Step-by-step instructions
1
Disable direct root login
2
Disable password-based SSH authentication
3
Enforce key-based SSH authentication
4
Restrict SSH access to approved users and groups
5
Configure sudo for least privilege — no unrestricted sudo
6
Remove shared or unused accounts
Validation & verification
Root login disabled
SSH accepts key-based authentication only
sudo usage logged and visible
Rollback & troubleshooting
Use break-glass credentials if access is lost
Test SSH key authentication before closing the session
Common errors & tips
Always test key-based auth before disabling password auth — avoid lockouts
Review and rotate SSH keys regularly — remove orphaned keys immediately
