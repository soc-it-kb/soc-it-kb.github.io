Tier-1 Quick Checklist

EM-03: Malicious Attachment / Malware Delivery

Use this checklist to triage an email with a suspicious attachment. Whether the user opened it determines the escalation path — always ask the user directly.

📧 Email threats Tier 1 Published 20 Dec 2025
📎 High-risk attachment types — these require immediate escalation if opened
.exe .dll .ps1 .iso .img .lnk .docm .xlsm .html .zip .pdf
🔔 When to use this checklist
Email gateway alert for a malicious or suspicious attachment / user report / EDR alert linked to an email delivery event.

Step 3 — Apply decision logic
🔴
YES to question 3 or 4 — Escalate to Tier-3 immediately. Treat as endpoint compromise.
⚠️
No execution but malicious indicators present — Escalate to Tier-2 with all evidence attached.
Step 4 — Mandatory actions
Preserve the original email and attachment — do not delete
Ask the user directly and explicitly whether they opened or executed the file
Search for other recipients of the same email
Attach all evidence to the incident ticket
Do NOT open or forward the attachment — handle in sandbox only
⚠️ Common Tier-1 mistakes to avoid
Assuming "blocked by gateway" means no risk — the attachment may have reached some users before the block was applied
Not confirming execution with the user — the user may have opened it and not mentioned it
Treating password-protected attachments as benign — passwords are used specifically to bypass email scanning
Delaying escalation after an EDR alert — an EDR alert means execution has already occurred
💡
Analyst Tip
If an attachment was opened, always assume endpoint compromise until EDR proves otherwise. Do not wait for confirmation — escalate immediately and let Tier-3 investigate.
