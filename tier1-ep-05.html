EP-05: USB / Removable Media Alert

Answer each question YES or NO and apply the decision logic. Escalate with evidence attached — do not reboot or alter the endpoint before escalating.

💻 Endpoint threats Tier 1 Published 28 April 2026
🔔 When to use this checklist
EDR alert, DLP notification, or user report of a USB or removable media device connected to a Windows endpoint — particularly where malware execution, data access, or policy violation is suspected.

Step 2 — Apply decision logic
✅
USB connected, no malware, no policy violation, known user — Document, remind user of USB policy, monitor, close.
⚠️
YES to questions 3, 4, 6, or 7 — Escalate to Tier-2. Secure the USB device. Follow EP-WI-USB-01 for safe handling.
🔴
YES to question 1, 2, or 5 — Escalate to Tier-3 immediately. Invoke EP-PB-01. Isolate endpoint. Do not reuse the USB on any other system.
🔴
YES to question 8 — Notify HR/Legal. Preserve chain of custody. Do not allow user access to the USB.
Step 3 — Capture evidence before escalating
Endpoint hostname and assigned user
USB make, model, and serial number if visible
EDR alert details and timeline
Files accessed or executed from the USB
Evidence of data copied to or from the USB
Physical location and circumstances of USB discovery
💡
Analyst Tip
Never open files from a suspicious USB to 'see what's on it' — this executes the content and potentially infects the analysis machine. Always physically label and secure the USB before doing anything else. The USB is evidence — treat it with the same care as any other forensic artefact.
