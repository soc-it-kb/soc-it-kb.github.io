ID-01: Suspicious Authentication Activity

Use this checklist to triage an authentication alert. Work through each question in order and follow the decision logic at the bottom.

🔐 Identity Tier 1 Published 20 Dec 2025
🔔 When to use this checklist
SIEM Alert / Identity Protection alert / MDR escalation indicating abnormal authentication behaviour.

Step 3 — Apply decision logic
✅
All answers = NO — Document your findings and close the ticket.
⚠️
Any answer = YES — Escalate to Tier-2 with all evidence attached.
🔴
YES to questions 5, 6, or 7 — Mark as High Priority and escalate immediately. Do not wait.
Step 4 — Mandatory actions before escalating
Capture and attach to ticket
User UPN
Timestamp(s) of the suspicious login
Source IP address(es)
Geo-location of the source IP
Device ID (if available)
MFA result — challenged / bypassed / not applied
Check and note in ticket
Recent password reset in last 7 days
Known travel or documented VPN use
Link to ticket
Related phishing ticket (if one exists)
MDR case reference (if applicable)
⚠️ Common Tier-1 mistakes to avoid
Assuming VPN usage without evidence — always verify before noting it
Closing an alert because "MFA is enabled" — check whether it was actually challenged and passed
Ignoring legacy authentication alerts — legacy auth bypasses MFA entirely
Treating VIP or admin logins as normal noise — these always warrant closer attention
Escalating without attaching the authentication evidence — Tier-2 cannot investigate without it
💡
Analyst Tip
If MFA is enabled and no MFA challenge is recorded in the logs, always assume increased risk and escalate — do not close.
