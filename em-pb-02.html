EM-PB-02: Malicious Attachment Incident Response

Structured decision framework for detecting, analysing, and responding to malicious email attachments — ensuring safe handling and rapid containment.

📧 Email threats Tier 1 / 2 / 3 MDR integrated

👤 Author: Paulo Ferreira

🏷️ Category: Email Threat

Active
Section 1
Purpose

Provide a structured decision framework for detecting, analysing, and responding to malicious email attachments, ensuring safe handling and rapid containment.

Section 2
Scope

✅ In scope

  • Attachments delivered via email
  • MDR escalations involving suspicious files
  • User-reported suspicious attachments

❌ Out of scope

  • Malware detected post-execution (Endpoint playbooks apply)
Section 3
Trigger Conditions
  • Email gateway flags attachment
  • User reports suspicious file
  • MDR escalates attachment-based threat
Section 4
Initial Validation

Confirm all three conditions before proceeding:

  • Is the attachment unsolicited?
  • Is the file type high-risk?
  • Is the sender external or spoofed?
High-risk file types
.html .iso .zip .lnk .img .xlsm .docm

If NO to any → Close or downgrade
If YES to all → Proceed to analysis (Section 5)

Section 5
Classification Logic

Classify the attachment into one of the following categories:

✅ BenignNo malicious behaviour detected
⚠️ SuspiciousObfuscation present, uncommon file format
💀 MaliciousPayload execution confirmed, C2 communication observed
🔑 Credential HarvestingHTML forms, redirects to fake login pages
📦 DropperSecondary payload download initiated
Section 6
User Interaction Assessment

Determine the extent of user interaction:

  • Did the user open the attachment?
  • Did the user enable macros?
  • Did the user execute embedded content?

If interaction confirmed → escalate severity and invoke endpoint containment (EP-RB-01)

Section 7
Response Strategy

Based on classification and user interaction, execute the following actions:

Isolate attachmentRequired
Perform sandbox analysisRequired
Determine blast radiusRequired
Containment actionsConditional
Section 8
Runbook Invocation Matrix

Invoke the relevant execution runbooks based on the scenario:

Attachment analysis EM-RB-03 › Email removal from mailboxes EM-RB-04 › Block malicious file hash EM-RB-06 › Account containment ID-RB-01 › Endpoint investigation EP-RB-01 ›
Section 9
Closure Criteria

An incident may be closed when all of the following are confirmed:

  • Attachment verdict recorded
  • Containment actions completed
  • User notified
  • IOCs shared with threat intelligence platforms
Section 10
Evidence to Capture
File hashes
Sandbox report
Email artefacts
User interaction confirmation
