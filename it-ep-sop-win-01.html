IT-EP-SOP-Win-01: Endpoint Security Baseline — Windows

Minimum recommended security baseline for all Windows-based endpoints — ensuring every device is deployed and operated in a secure-by-default state.

Section 1
Purpose

This SOP defines the minimum recommended security baseline for all Windows-based endpoints. The objective is to ensure that every Windows device is deployed and operated in a secure-by-default state, reducing attack surface, limiting lateral movement, and enabling effective detection, response, and recovery.

📌 This SOP defines what controls should be in place. Detailed implementation steps are in the associated Work Instructions (IT-EP-WI-Win-01 through Win-05).

Section 2
Scope
  • All Windows workstations and laptops
  • All institution-managed and institution-owned Windows endpoints
  • All users including standard and privileged users

Out of scope: Windows Servers (separate SOP), BYOD devices unless explicitly approved.

Section 3
Baseline Principles

📋 CIS Level 1 minimum

Default baseline — universally recognised and audit-ready

🎯 High-impact, low-disruption first

Controls that deliver security value without operational burden

🔐 Least privilege

For users and administrators — standard user by default

🛡️ Native security first

Augmented by enterprise tooling, not replaced by it

Section 4
Minimum Security Requirements

💻 OS Configuration

  • Supported Windows version only
  • Secure Boot and TPM enabled
  • Automatic updates enabled or centrally managed
  • Unused services and legacy protocols disabled (e.g. SMBv1)

🔐 Identity & Access

  • Standard users by default
  • Local administrator access restricted
  • No shared local administrator accounts
  • Authentication policies aligned with institutional standards

🛡️ Endpoint Protection

  • EDR agent installed and active
  • ASR baseline rules enabled
  • Exploit protection enabled
  • Tamper protection enabled — not disableable by end users

🔒 Disk Encryption

  • BitLocker with TPM-based key protection
  • Recovery keys securely escrowed
  • Removable media restricted or controlled

📊 Logging & Telemetry

  • Security event logging enabled
  • PowerShell script block and module logging
  • EDR telemetry accessible to SOC

👤 User Configuration

  • Screen lock with inactivity timeout
  • Strong authentication enforced
  • Browser security configuration applied
  • Macro execution restricted by default
