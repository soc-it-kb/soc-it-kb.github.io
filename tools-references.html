Framework reference: MITRE ATT&CK Enterprise Mitigations  ·  Status: Authoritative — lists ATT&CK mitigations (M-codes) materially implemented by the KB’s preventive content. Detection coverage is captured in the tactic cards above; this section captures structural defences.
MitigationNameImplemented byNotes
M1018User Account ManagementIT-ID-SOP-01, IT-ID-WI-01, IT-ID-WI-02JML lifecycle, recertification cadence per tier, authoritative-source rule
M1026Privileged Account ManagementIT-ID-WI-05, IT-SI-WI-04, IT-EP-SOP-PAW-01Time-bound eligibility, two-person approval at Tier-0, PAW separation, PAM vault
M1027Password PoliciesIT-CL-WI-01, IT-SRV-WI-DC-01Tenant identity baseline + AD policy; deprecation of legacy auth
M1032Multi-factor AuthenticationIT-CL-WI-01, IT-ID-WI-05FIDO2 / WHfB / cert-based MFA for Global Administrators; phishing-resistant required for Tier-0
M1036Account Use PoliciesIT-ID-WI-04, IT-ID-WI-05Standing-privilege minimisation; non-human identity governance; sign-in anomaly detection on service accounts
M1053Data BackupGRC-DR-SOP-01, GRC-DR-WI-01, GRC-FRM-DR-01Immutable + isolated backup as Tier-1 minimum; tested restore cadence; 3-2-1 baseline as floor not ceiling

Mitigations are listed where the KB has explicit, named preventive content implementing them. Many other ATT&CK mitigations (network segmentation, application allowlisting, audit logging) are addressed across multiple baselines — this list captures the ones with a single authoritative source.

Primary framework — adopted standard
NIST Cybersecurity Framework 2.0
The globally adopted voluntary framework used as the primary reference structure for all content in this knowledge base. Released February 2024. Applicable to organisations of any size, sector, or geography — including NGOs, not-for-profits, and SMBs.
6 Core Functions 22 Categories 106 Subcategories Govern · Identify · Protect · Detect · Respond · Recover

Continuous monitoring, adverse event analysis
