EP-03: Ransomware Indicators

Answer each question YES or NO and apply the decision logic. Escalate with evidence attached — do not reboot or alter the endpoint before escalating.

💻 Endpoint threats Tier 1 Published 28 April 2026
🔔 When to use this checklist
EDR alert, user report, or monitoring detection suggesting ransomware activity on a Windows endpoint — including file encryption, ransom notes, or volume shadow copy deletion.

Step 2 — Apply decision logic
🔴
YES to ANY question 1–7 — This is a ransomware incident. Invoke EP-PB-05 immediately. Isolate endpoint via EP-WI-02. Escalate to Tier-3. Do NOT reboot.
⚠️
Suspicious indicators but no confirmed encryption — Escalate to Tier-2 immediately. Isolate endpoint as a precaution. Do not wait for confirmation.
✅
All NO — false positive suspected — Document findings, escalate to Tier-2 for review, monitor closely.
Step 3 — Capture evidence before escalating
Endpoint hostname and user
Screenshots of renamed files and ransom note if present
EDR alert details including process tree
List of affected file paths and extensions
Network share and mapped drive access details
Isolation timestamp
💡
Analyst Tip
With ransomware, speed of isolation is everything. Do not wait for Tier-2 approval before isolating — isolate immediately and notify in parallel. Every second of network connectivity risks spreading to file shares and other endpoints. When in doubt, isolate first and ask questions after.
