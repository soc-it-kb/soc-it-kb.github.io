EP-RB-03: Remove Malware & Persistence Artefacts

Remove malware binaries, scripts, and all persistence mechanisms from a compromised endpoint to restore system integrity — followed by reboot validation to confirm clean status.

💻 Endpoint threats Tier 2 / 3 Eradication

Section 1
Purpose

Remove malware binaries, scripts, and persistence mechanisms from a compromised endpoint to restore system integrity.

This runbook assumes the endpoint is already isolated (EP-RB-02), or the risk of execution has been fully mitigated.

Section 2
Required Access
🛡️ EDR remediation privileges 📊 Endpoint telemetry access 🔑 Administrative OS permissions (if required)
Section 3
Inputs Required
🖥️ Endpoint hostname 📎 Identified malicious files / artefacts 🔒 Persistence indicators from EP-RB-01 🎫 Incident reference ID
Section 4
Step-by-Step Procedure
1
Validate Artefacts to Remove
1
Review investigation findings from EP-RB-01
2
Confirm the following for each artefact before removal:
  • File paths confirmed
  • Hashes verified as malicious
  • Persistence locations identified
3
Ensure all artefacts are confirmed malicious before proceeding — do not remove based on suspicion alone
2
Remove Active Processes
1
Terminate all identified malicious processes via the EDR console
2
Confirm terminated processes do not respawn — monitor for 2–3 minutes
Respawning indicates persistence is still active — complete Phase 4 before re-attempting process termination
3
Capture process termination evidence in the EDR console
3
Remove Files and Scripts
1
Quarantine or delete all malicious binaries via the EDR
2
Remove dropped scripts, loaders, or secondary payloads
3
Confirm each file removal was successful — check file system for any remaining copies
4
Remove Persistence Mechanisms

Remove and confirm each of the following persistence types found during investigation:

Scheduled tasks
Registry autorun keys
Malicious services or drivers
Startup folder items
WMI event subscriptions
5
Reboot and Re-Validate
1
Reboot the endpoint if safe to do so — confirm with IT operations if needed
2
Monitor post-reboot telemetry in the EDR for any re-emergence of malicious activity
Some persistence mechanisms only trigger on reboot — this step is critical
3
Confirm no persistence has been recreated after the reboot
6
Document Actions
1
Record in the incident ticket:
  • All artefacts removed (paths, hashes)
  • All persistence mechanisms removed
  • Reboot status and post-reboot validation result
2
Attach EDR screenshots and evidence exports to the ticket
Section 5
Expected Outcome

✅ All malware fully removed — no malicious processes running
✅ All persistence mechanisms eliminated
✅ Endpoint stable post-reboot with EDR reporting clean status

💡 When to rebuild instead: If confidence in complete eradication is low, if the malware family is complex, or if the endpoint hosted sensitive data — a full rebuild from a known-good baseline is the safer option.

Section 6
Validation Checks
  • No malicious processes running post-reboot
  • No persistence mechanisms recreated after reboot
  • EDR reports clean status for the endpoint
Section 7
Common Errors & Tips
Removing malware files without removing persistence leads to reinfection — always complete Phase 4 before declaring success
Some persistence mechanisms only trigger on reboot — Phase 5 reboot validation is not optional
If confidence in complete eradication is low, rebuild the endpoint from a known-good baseline — remediation is faster but riskier
Section 8
Output Artefacts
Removal confirmation
Artefact list (paths, hashes)
Reboot validation result
