🔒 Security Baselines

Minimum recommended security configurations for all endpoints and servers. The secure-by-default foundation that limits attack surface before any vulnerability exists — bridging Cyber Security and IT Operations.

CIS Benchmarks Level 1 NIST CSF 2.0 — Protect ISO 27001 Tool-agnostic
Section 1
Purpose

The Security Baselines domain defines the minimum recommended security configurations that should be applied to all endpoints and servers. These baselines ensure that devices are deployed and operated in a secure-by-default state, reducing risk, limiting attack surface, and supporting rapid incident response and recovery.

This domain bridges Cyber Security and IT Operations, providing clear, actionable guidance that local, regional and global IT teams can consistently apply regardless of location or maturity level.

Section 2
Scope

💻 Applies to

  • All institution-managed endpoints (workstations and laptops)
  • All on-premises and cloud-hosted servers
  • All users with administrative or privileged access

📋 Compliance expectation

Unless explicitly approved through the exception process, all devices must comply with the defined minimum baselines. Unapproved deviations are considered non-compliant.

Section 3
Baseline Principles

🎯 Minimum viable hardening

Focused on high-impact, low-risk controls that can be applied broadly without operational disruption

📋 CIS-aligned

Based on CIS Benchmarks Level 1 as the default minimum — universally recognised and audit-ready

🔧 Tool-agnostic

SOPs define what must be implemented — Work Instructions define how, per platform and tooling

🌍 Operationally realistic

Designed for distributed and resource-constrained environments at any maturity level

🛡️ SOC-supportive

Enables detection, response, and recovery activities through consistent telemetry and configuration

🔄 Living documents

Evolve with the threat landscape, incident learnings, and CIS benchmark updates

Section 4
Baseline Domains

💻 Endpoint Security Baselines

  • Windows Workstations
  • macOS Workstations (where applicable)
  • Linux Workstations (where applicable)
  • Privileged Access Workstations (PAW)

Covers: OS hardening · Local administrator controls · Endpoint protection · Disk encryption · Logging and telemetry

🖥️ Server Security Baselines

  • Windows Servers
  • Linux Servers
  • Domain Controllers (hardening addendum)
  • Virtualisation hosts and critical infrastructure

Covers: OS hardening · Service exposure reduction · Administrative access controls · Logging and monitoring

Section 8
Validation, Exceptions & Governance

✅ Validation

Configuration verification checklists, automated or manual validation steps, and SOC or IT-led spot checks. Evidence retained for audit.

⚠️ Exceptions

Where a control cannot be implemented: document justification, record compensating controls, obtain approval, set review date. See IT-GOV-SOP-01.

🔄 Change management

Baselines are living documents. Changes tracked in the Baseline Change Log and communicated to stakeholders. Annual review minimum.
