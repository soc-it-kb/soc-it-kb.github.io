EM-RB-06: Block Malicious File Hash

Block a confirmed malicious file hash across email, endpoint, and security platforms to prevent execution or re-delivery across the environment.

📧 Email threats Tier 1 / 2 Multi-platform
📋 ID: EM-RB-06

👤 Author: Paulo Ferreira
·
📅 Published: 22 December 2025
·
🏷️ Category: Email Threat
·
Active
Section 1
Purpose

Block a confirmed malicious file hash across email, endpoint, and security platforms to prevent execution or re-delivery.

Section 2
Required Access
🛡️ EDR console (CrowdStrike / equivalent) 📧 Email security platform 🔍 Threat Intelligence platform 🎫 SOC ticketing system
Section 3
Inputs Required
#️⃣ File hash (SHA256 preferred) 🔍 Source of detection 🏷️ Associated campaign (if known)
Section 4
Step-by-Step Procedure
1
Validate the hash format — confirm it is SHA256 where possible
SHA256 is preferred over MD5 or SHA1 to avoid hash collisions
2
Confirm malicious verdict via sandbox report or Threat Intelligence platform
Do not block without a confirmed verdict — check VirusTotal or internal TI
3
Add the hash to the EDR prevention policy to block execution on all endpoints
4
Add the hash to the email security platform block list to prevent re-delivery
5
If applicable, add to the SIEM watchlist for future detection visibility
6
Verify policy propagation — confirm the block is active across all platforms
7
Document the action in the incident ticket including hash, verdict source, and platforms blocked
Platform blocking coverage

EDR / XDR

Blocks file execution on all managed endpoints

Email security

Prevents re-delivery of the file via email

SIEM watchlist

Flags any future hash encounters for detection

Section 5
Expected Outcome

✅ File execution blocked on all managed endpoints
✅ Email re-delivery of the hash prevented
✅ Hash visible in SIEM for future detection

Section 6
Validation Checks
  • Attempted execution is blocked and logged in EDR
  • No further email deliveries of the hash observed
Section 7
Common Errors & Tips
Do not block hashes without a confirmed malicious verdict — false positives can block legitimate software
Always prefer SHA256 over MD5 or SHA1 to avoid hash collisions blocking the wrong file
Coordinate with MDR before applying global blocks across the entire environment
Section 8
Output Artefacts
Block rule references (EDR, email, SIEM)
Verdict confirmation source
